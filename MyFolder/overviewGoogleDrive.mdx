Google drive dataset overview

1. Purpose of the Dataset

The Vectrix Google Drive dataset represents the internal document storage used by the Vectrix team.

It contains a collection of 35 files that simulate how a startup stores and organizes documentation.

The dataset includes technical specifications, meeting notes, spreadsheets, contracts, presentations, and planning documents.

It is designed to test document retrieval, version resolution, and cross-document reasoning for AI systems.

4. Version Conflicts

Some documents exist in multiple versions with conflicting information. These are intentionally included to simulate real company documentation updates.

Examples include:

Architecture Specification

v1 uses TimescaleDB

v2 migrates to ClickHouse

Pricing Model

v1 uses Pro tier ($29/month)

v2 introduces Growth tier ($19/month)

Data Retention Policy

v1 retains data for 180 days

v2 introduces tiered retention (90 days raw, 2 years aggregated)

Senior Frontend Job Description

v1 salary band: $140K–$155K

v2 salary band: $145K–$165K

These version conflicts reflect real product and business decisions made by the company.

5. Edge Cases Included

The dataset includes several realistic irregularities that appear in real company documentation.

Examples include:

A contract with visible tracked changes and legal redlines

A spreadsheet containing a formula error

A presentation marked as “DRAFT – DO NOT DISTRIBUTE”

A document with unfinished sections marked as [TBD]

A meeting notes file containing only the meeting agenda with no notes

These cases simulate common documentation issues found in real organizations.

6. Cross-Document References

Many documents reference information from other internal tools used by Vectrix.

Examples of references include:

Slack discussions about engineering decisions

GitHub pull requests related to architecture changes

Linear issues tracking feature development

Email conversations with customers or partners

These references create connections between documents and other datasets in the system.

7. Notion Migration Artifact

Vectrix previously used Notion for documentation but later migrated most content to Google Drive.

As part of the dataset:

One technical document still exists in Notion format

The Notion file includes metadata stored in a notion_manifest.json

The Drive version of the document is incomplete, reflecting a partial migration

This simulates the common problem of documentation split across multiple platforms.

8. Dataset Use Cases

The Google Drive dataset can be used to test AI systems on tasks such as:

Finding the latest version of a document

Resolving conflicting information across versions

Linking information across multiple internal tools

Understanding structured and unstructured documents

Identifying incomplete or outdated documentation
